Methodology

Phase-by-phase workflow used across managed programs. Click each phase to expand commands.

01 Subdomain Reconnaissance ▶

Map the full subdomain surface before touching any application. Combine passive and active sources, then resolve and probe for live hosts.

# Passive enumeration
subfinder -d target.com -o subfinder_output.txt
amass enum -passive -d target.com -o amass_subdomains.txt

# Certificate transparency
curl -s "https://crt.sh/?q=%.target.com&output=json" \
  | jq -r '.[].name_value' | sort -u > ct.txt

# Merge and resolve
cat subfinder_output.txt amass_subdomains.txt ct.txt \
  | sort -u | dnsx -silent -o live_subdomains.txt

# Probe for live web
httpx -l live_subdomains.txt -title -status-code -o all_subdomains.txt
02 API & Endpoint Discovery ▶

Extract endpoints from JavaScript files, Wayback Machine, and GAU. Use ffuf for directory brute-force against in-scope targets.

# Historical URLs
gau target.com | tee gau_urls.txt
waybackurls target.com >> wayback_urls.txt

# JS endpoint extraction
cat js_urls.txt | while read url; do
  python3 LinkFinder/linkfinder.py -i "$url" -o cli 2>/dev/null
done | sort -u > api_endpoints.txt

# Secret scanning in JS
python3 SecretFinder/SecretFinder.py \
  -i https://target.com/app.js -o cli

# Directory fuzzing (in-scope only)
ffuf -w SecLists/Discovery/Web-Content/api/api-endpoints.txt \
  -u https://target.com/api/FUZZ \
  -mc 200,201,301,302,401,403 -o ffuf_backup.json
03 SSO / OAuth Testing ▶

Test Keycloak, Okta, and custom SSO for common misconfigurations. Always include the program tracking header per program rules.

# Keycloak realm enumeration
curl -s https://sso.target.com/auth/realms/master \
  -H "X-BUGBOUNTY: zwanski" | jq .

# OIDC discovery
curl -s https://sso.target.com/auth/realms/TARGET/.well-known/openid-configuration

# redirect_uri bypass (double-encode)
# Test: redirect_uri=https://attacker.com%2523@target.com/callback

# Dynamic Client Registration probe
curl -s -X POST https://sso.target.com/auth/realms/TARGET/clients-registrations/openid-connect \
  -H "Content-Type: application/json" \
  -d '{"redirect_uris":["https://attacker.com"]}'
04 PoC & Impact Demonstration ▶

Convert triaged findings into accepted submissions by demonstrating concrete, minimal-impact proof of concepts. No broad scanning — targeted validation only.

# API key validation (minimal impact)
curl -s -X GET "https://api.target.com/v1/me" \
  -H "Authorization: Bearer EXPOSED_KEY" \
  -H "X-BUGBOUNTY: zwanski" | jq '{id,email,role}'

# Info disclosure — capture only non-sensitive fields
curl -s https://target.com/api/serverinfo \
  | jq '{version, environment, build_date}'

// 03 · Setup

Installation

Clone the repo and install dependencies. Tested on Kali Linux.

BASH 
# Clone the repository
git clone git@github.com:zwanski2019/zwanski-Bug-Bounty.git
cd zwanski-Bug-Bounty

# Python dependencies
pip install -r requirements.txt

# Run the recon toolkit (Streamlit)
streamlit run bb_recon_tool.py

# Make shell scripts executable
chmod +x *.sh
./verify_tools.sh   # check tool availability
REQUIREMENTS 
# Core recon tools (apt)
sudo apt install subfinder amass dnsx httpx ffuf nuclei nmap

# Go tools
go install github.com/lc/gau/v2/cmd/gau@latest
go install github.com/tomnomnom/waybackurls@latest

# Python tools
git clone https://github.com/GerbenJavado/LinkFinder
git clone https://github.com/m4ll0k/SecretFinder
